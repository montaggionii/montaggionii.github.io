01 · ARQUITECTURA SPRING BOOT

Capas con responsabilidades delimitadas.

La evidencia disponible en FidelyFood separa controllers, services y acceso a datos con JPA/Hibernate. Esta organización reduce acoplamiento y permite concentrar validaciones y reglas de negocio en servicios.

02 · JWT AUTHENTICATION

La sesión atraviesa frontend y API con control de acceso.

El patrón documentado combina autenticación, token JWT, interceptor del cliente y autorización de endpoints. La comprobación de permisos debe permanecer en backend aunque la interfaz oculte acciones no permitidas.

03 · APIS REST

Contratos claros antes de conectar pantallas.

Recursos y respuestas

Endpoints consistentes, códigos HTTP adecuados y estructuras de error comprensibles facilitan el consumo desde Angular.

Documentación

OpenAPI permite revisar el contrato y probar operaciones sin convertir la documentación en una promesa de cobertura total.

04 · ANGULAR ARCHITECTURE

Rutas, guards e interceptor como fronteras de cliente.

El frontend puede organizar experiencias por rol mediante rutas lazy, guards y un interceptor para adjuntar el token. Estos controles mejoran la experiencia; no reemplazan la autorización aplicada en la API.
